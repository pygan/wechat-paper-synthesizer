# 信源画像

> **版本**: v1.0  
> **用途**: 记录已知解读信源的风格特征、偏向、盲区和使用历史，让 Skill 从"无状态工具"进化为"有记忆的系统"。  
> **更新规则**: 每次完成六维度报告后，评估本次使用的信源，更新或新增画像条目。每次更新在条目末尾标注日期。

---

## 使用指南

在 Skill 运行时，按以下优先级使用信源画像：

### Phase 0：加载画像

1. 读取本文件，获取所有已知信源的画像
2. 识别与当前论文领域/类型最匹配的信源及其特征（供 Phase 5 合成时参考）
3. **不限制 Phase 1 搜索范围**：画像仅作用于 Phase 5 合成阶段（信任权重分配、盲区补偿），Phase 1 全源搜索不设域名过滤器

### Phase 5：批判性使用

根据画像特征调整对各信源信息的信任权重：

| 信息类型 | 优先信任的信源特征 | 需谨慎对待的信源特征 |
|----------|-------------------|---------------------|
| 技术细节 | 高深度、课题组/专业媒体 | 低深度、泛媒体 |
| 量化数据 | 高深度、有定量习惯 | 定性概括型、产业媒体 |
| 批判观点 | 高批判性、学术媒体 | 低批判性、官媒 |
| 产业判断 | 强中国视角、投资/产业媒体 | 纯学术媒体 |
| 宏观政策 | 官媒、强中国视角 | 纯技术媒体 |

### Phase 6：更新画像

报告完成后，评估本次使用的信源：
- 已有画像的信源：更新使用次数、最近使用日期，如果发现新特征则补充
- 首次出现的信源：创建新画像条目

## 中文科技解读信源画像

### DrugAI（腾讯云开发者社区）

- **URL 模式**: `cloud.tencent.com/developer/article/*`
- **类型**: AI+制药专业媒体
- **风格**: 学术技术流，侧重方法细节和架构分析
- **技术深度**: ★★★★★
- **批判性**: ★★★★☆
- **乐观程度**: 审慎乐观
- **中国视角**: 弱
- **擅长**: 技术架构拆解、方法流程还原、关键实验数据
- **盲区**: 不覆盖商业化落地和产业视角；对失败案例和限制条件涉及较少
- **首次记录**: 2026-05-20
- **最近使用**: 2026-05-26（Robin, Ghareeb et al., Nature 2026）
- **使用次数**: 2
- **进化记录**: Robin 分析中再次确认其技术深度优势——是唯一给出 ripasudil 1.89× 和 ABCA1 3× 精确数字的信源。保持了技术细节最全面的一贯风格。

### ScienceAI（搜狐）

- **URL 模式**: `www.sohu.com/a/*`
- **类型**: 学术科技媒体
- **风格**: 学术对比流，习惯将同领域多篇论文并置对比
- **技术深度**: ★★★★☆
- **批判性**: ★★★☆☆
- **乐观程度**: 乐观
- **中国视角**: 弱
- **独特价值**: 双系统/多论文横向对比视角（如 Co-Scientist vs Robin），提供竞品信息
- **盲区**: 批判性中等，不以质疑为主要立场；产业商业视角较弱
- **首次记录**: 2026-05-20
- **最近使用**: 2026-05-26（Robin, Ghareeb et al., Nature 2026）
- **使用次数**: 2
- **进化记录**: Robin 分析中再次展示对比视角价值（Robin vs Co-Scientist 同日 Nature），独家提供了 hallucination 数据（o4-mini 替换 Crow = 45% 幻觉率）。批判性与 DrugAI 相近，技术细节深度略逊。

### 智药局（新浪财经）

- **URL 模式**: `finance.sina.com.cn/*`
- **类型**: 产业/商业媒体
- **风格**: 产业商业流，侧重商业化落地和投资逻辑
- **技术深度**: ★★★☆☆
- **批判性**: ★☆☆☆☆
- **乐观程度**: 非常乐观
- **中国视角**: 强（商业化落地、投资赛道判断）
- **独特价值**: 投资人视角、"谁在做、钱在哪"信息、产业落地判断
- **盲区**: 技术细节浅；倾向于正面叙事，几乎不报道失败案例；量化数据不精确
- **首次记录**: 2026-05-20

## 信源组合策略

基于已知信源画像，推荐的信源组合：

### 标准组合（≥3 篇）

| 角色 | 推荐信源 | 提供什么 |
|------|----------|----------|
| 技术深度担当 | DrugAI | 架构拆解、方法还原、实验数据 |
| 横向对比视角 | ScienceAI | 竞品信息、多系统对比 |
| 实验验证细节 | 生物世界（163 或 CPHI） | 效率数据、补充实验信息 |
| 产业落地判断 | 智药局 | 投资判断、商业化路径 |

### 增强组合（≥4 篇，在标准组合基础上增加）

| 补充角色 | 推荐信源 | 提供什么 |
|----------|----------|----------|
| 实验验证细节 | 生物世界 | 补充实验数据 |
| 政策宏观背景 | 科技日报 | 政策视角 |

### 信源不足时的优先级

如果无法获取足够的已知信源：
1. 优先找 **DrugAI**（技术深度最高，还原原文能力最强）
2. 其次 **ScienceAI**（对比视角提供竞品信息）
3. 再次 **智药局** 或 **生物世界**（验证数据+产业判断互补）

## 系统性偏向说明

基于已有分析，中文科技解读文章存在以下**系统性偏向**（非个别信源问题，而是整个信源生态的特征）：

| 偏向类型 | 表现 | 对报告的影响 |
|----------|------|-------------|
| **正面叙事偏好** | 报道成功案例，忽略失败案例 | 报告中可能遗漏论文中的无效结果和失败实验 |
| **量化松弛** | 用"高分""优秀"替代精确数字 | 报告中可能缺失具体评分、精确数据 |
| **突破叙事** | 强调"首次""突破"，淡化渐进性 | 报告对论文实际贡献水平的判断可能偏乐观 |
| **技术细节跳过** | 不覆盖UGT1A1代谢、TP53突变等纯技术细节 | 报告中可能缺失论文最精彩的推理链 |
| **竞品淡化** | 不报道或不强调竞品表现 | 报告对相对优势的判断可能偏乐观 |
| **自我警示忽略** | 不引用论文Discussion中的谦逊声明 | 读者可能不知道论文作者自己的谨慎态度 |
| **评估方法遗漏** | 不报道论文判断自身方法好坏的标准和指标体系（如 Co-Scientist 的 Plausibility/Novelty/Testability/Safety/Alignment 五大门控标准，Robin 的 BixBench/Finch/LLM judge 评估框架） | 报告可能描述了方法功能，但缺失了论文的设计哲学——"凭什么说这个方法好" |

此偏向说明已纳入报告的免责声明中，提醒读者注意。
